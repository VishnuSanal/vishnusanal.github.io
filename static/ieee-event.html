The Four Freedoms

  • Freedom 0: Run the program for any purpose
  • Freedom 1: Study how it works & modify it
  • Freedom 2: Redistribute copies
  • Freedom 3: Distribute your modified versions


Open Source licenses protect these freedoms

Making a Real-World Contribution

Types of Contributions

It's not just code!

  • Documentation / Guides
  • Design / UX - feedbacks too
  • Bug reports & triaging issues
  • Translations, localization, and accessibility
  • Community support and activity

Finding a Repo

  • GitHub Trending — discover popular projects using language & stack sort
  • Labels: good-first-issue, low-hanging-fruit, easy, etc.
  • goodfirstissue.dev — beginner-friendly issues aggregator
  • Most viable: something you already use!


Fix a bug that annoys you — that's the best motivation

Contribution Guidelines

  • Refer to the project's CONTRIBUTING.md
  • Read GitHub's general contribution guidelines
  • Check issue labels, issue/PR templates, and coding standards

The Contribution Workflow

  1. Research — build locally, understand the codebase
  2. Discuss — comment on the issue, join the community rooms, and get assigned (if necessary)
  3. Code — start working & send a PR!
  4. Review — respond to feedback, iterate
  5. Merge — celebrate!

PR Reviews — Do's & Don'ts

Do

  • Be polite and patient
  • Explain your changes clearly
  • Follow the templates strictly
  • Request reviews proactively
  • Iterate based on feedback

Don't

  • Spam maintainers for reviews
  • Be impatient
  • Take feedback personally
  • Submit massive, unreviewable PRs
  • Ignore CI failures

Keep Progressing

  • Start small, grow your confidence
  • Move from docs fixes to bug fixes to features
  • Become a regular contributor
  • Regular Contributions => Maintainer Invitation!

Starting Your Own OSS Project

A checklist

Project Foundations

  • README.md is the heart of your project
  • License your project
  • Have CONTRIBUTING.md file & CODE_OF_CONDUCT.md file
  • Create PR & Issue templates
  • Label your issues properly
  • Make it welcoming to contributions

Community & Quality

  • Engage via issue threads & GitHub discussions
  • Use linters & code quality checks
  • Add relevant topics/tags on GitHub
  • Deploy docs with GitHub Pages (if applicable)
  • Set up CI/CD (GitHub Actions, etc.)
